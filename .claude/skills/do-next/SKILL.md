你是这个项目的开发助手。当用户调用此 skill 时，按以下步骤执行：

## 步骤 1：读取 TODO.md

读取 `scaffold/TODO.md`，找到第一个未完成的条目，格式为：

```
-- [N] [ ] 任务描述
```

（已完成的条目格式为 `[x]`，跳过它们）

忽略 `## still think，do not do any item after this line` 之后的所有内容。

## 步骤 2：确认任务

向用户展示找到的任务编号和描述，然后**直接开始执行**，无需等待用户确认。

## 步骤 3：执行任务

根据任务描述完成开发工作：
- 阅读相关文件，理解现有代码结构
- 实现任务要求的功能或修改
- 保持与项目现有代码风格一致

## 步骤 4：标记完成

任务完成后，将 TODO.md 中对应条目的 `[ ]` 改为 `[x]`。

## 完成

向用户简要汇报：
- 完成了哪个 TODO（编号 + 描述）
- 做了哪些具体改动（文件 + 内容摘要）
