# 产品需求文档: AI 实时翻译

## 1. 引言

本文档概述了“AI 实时翻译”网络应用的产品需求。该应用将为用户提供中英文之间的实时语音翻译。

## 2. 目标

*   为用户提供无缝、直观的实时翻译体验。
*   准确捕捉语音并提供高质量的翻译。
*   允许用户整理和查看他们的翻译历史。
*   提供安全可靠的服务。

## 3. 用户画像

*   **学生**: 正在学习一门新语言（英语或中文）的学生，希望练习发音并获得即时反馈。
*   **商务人士**: 需要与说不同语言的同事或客户沟通的专业人士。
*   **游客**: 到外国旅游需要基本交流帮助的游客。

## 4. 用户故事

*   作为用户，我希望能够对着麦克风说话，并在屏幕上看到实时翻译。
*   作为用户，我希望能够在中英文互译之间切换。
*   作为用户，我希望能够创建主题来整理我的翻译。
*   作为用户，我希望能够查看特定主题的过往翻译。
*   作为用户，我希望能够使用我的 Google 或 Hotmail 帐户登录应用程序。

## 5. 功能

### 5.1. 实时翻译
*   应用程序将使用浏览器的麦克风捕捉用户的声音。
*   应用程序将使用语音识别 API 将用户的语音转换为文本。
*   应用程序将使用 Gemini API 或 Kimi API 将文本翻译成目标语言。
*   翻译后的文本将以“生动”的方式逐句显示在屏幕上。

### 5.2. 主题管理
*   用户将能够创建、重命名和删除主题。
*   用户可以选择一个活动主题进行录制。一次只能有一个主题处于活动状态。
*   所有翻译都将保存到当前活动的主题中。

### 5.3. 翻译历史
*   用户可以查看所选主题的所有翻译历史。
*   历史记录将显示原始文本和翻译后的文本。

### 5.4. 用户认证
*   用户可以使用他们的 Google 或 Hotmail 帐户通过 OAuth 注册和登录。

## 6. 技术要求

*   **前端**: Next.js
*   **UI 组件库**: MUI (Material UI)，用于界面主题与组件设计
*   **后端**: Node.js
*   **缓存数据库**: Redis
*   **主数据库**: MongoDB
*   **实时通信**: WebRTC / WebSocket 用于访问麦克风和实时通信。
*   **API**:
    *   语音识别 API
    *   Gemini API/ Gemini live API / Kimi API 用于翻译。

## 7. 未来工作

*   支持更多语言。
*   离线模式。
*   能够编辑和分享翻译。
*   移动应用程序（iOS 和 Android）。
